Case study — OVO app

Redesigning filters in OVO's app

As offers and rewards grew, customers couldn't find the ones meant for them. This fixed that.

Role
Content design
Tools
Figma
Skills
IA & naming, usability testing, accessibility

Overview

As the number of offers and rewards inside OVO's app grew, the existing filter component started working against customers rather than for them: no active state on landing, unclear which filters were applied, and labels so long they wrapped to two lines.

Filtering had become something to fight rather than use. The component was also too tall for what it actually did — taking up real estate without earning it.

Process

Discover

Audited the existing component and reviewed how Spotify, Airbnb, and Deliveroo handle filtering — landing on three guiding principles: short, one-word labels; clear active states; and an "All" reset.

Define

Rewrote every filter label against those principles and created a naming ruleset so future filters stay consistent without another redesign.

Design

Iterated the layout — including an "All" default view — and refined the UI: icon alignment, spacing, and a pill-style active state.

Deliver

Ran unmoderated usability testing, deliberately excluding UX professionals, using neutral task-based questions, ahead of an A/B rollout.

Key screens & decisions

Results

25,000 customers in the first A/B rollout
3 guiding principles now used for every new filter
1 reusable component, ready for the rest of the app
"Simple" unprompted usability testing feedback

Why it worked

Building the naming ruleset alongside the redesign meant this wasn't just a one-off fix — it's a component the wider team can now reuse without reopening the same debates about labels and states, reducing design debt before it had the chance to build up.
